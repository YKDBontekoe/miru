# Security Policy

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Public issues are visible to everyone before the issue is fixed, which gives attackers advance notice.

Please report security vulnerabilities using [GitHub Private Security Advisories](../../security/advisories/new).

This keeps the report confidential between you and the maintainers until a fix is released.

### What to include

- A clear description of the vulnerability
- The affected component (backend API, frontend, database, auth, infrastructure)
- Steps to reproduce (if applicable)
- Potential impact if exploited
- Your suggested fix (optional)

### Response timeline

- **Acknowledgement:** within 48 hours
- **Triage and severity assessment:** within 5 business days
- **Fix and disclosure:** coordinated with you based on severity

We ask that you do not publicly disclose the vulnerability until a fix has been released.
